mm-datepicker

A single and dual (range) date/time picker component including numeric and calendar based inputs

Example HTML Editor

<script> window.addEventListener("polymer-ready", function() { document.querySelector("#t").open = function() { this.$.dp.open(); }; }); </script> <template is="auto-binding" id="t"> <mm-button id="btn" on-tap="{{open}}"> <label>Select a Date</label> </mm-button> <mm-datepicker id="dp" target="{{$.btn}}" useTimezone="false" useRange="false"> </mm-datepicker> </template>

Example Output

Attributes

Attribute Options
useTimeZone : boolean (Optional) Default value: true Enables timezone dropdown when true (requires external tz data) none
useRange : boolean (Optional) Default value: true Enables date range dropdown for preselected ranges none
useTime : boolean (Optional) Default value: true Enables input boxes for time selection none
useCommit : boolean (Optional) Default value: true Shows a footer which contains save/close buttons. Without pressing save the data is not persisted to the start/end binds none
resetOnClose : boolean (Optional) Default value: true When useCommit is true and resetOnClose is true, the picker will be reset to default values when closed if a user does not press the save button none
dual : boolean (Optional) Default value: true When true, the datepicker shows a start and end date panel for ranges none
dateFormat : string (Optional) Default value: MM/DD/YYYY moment.js format string used for date formatting--note that the input mask needs to be edited independently for format changes none
timeFormat : string (Optional) moment.js format string used for time formatting--note that markup currently only supports 12 hour time with a fixed dropdown. none

Public Methods

Method Arguements
reset(startend) Resets the selected date either to the start/end value or to the values specified in arguments start : Date (Optional) Default value: null A date object representing the start date (and time) to reset the picker instance to. end : Date (Optional) Default value: null A date object representing the end date (and time) to reset the picker instance to.
open() Shows the datepicker instance none
close() Closes the datepicker instance none
save() Behaves simliarly to close, excecpt that any pending values are commited (when none

Public Properties

Properties Options
rangeValue : string The currently selected range value for preset ranges none
rangePresets : Array Default value: null An array of objects which should contain a name, start and end property. Name is a string representing the preset's label. start and end should be JS Date objects that specify a valid start and end range to be selected. none
rangeDescription : string Default value: Select a Predefined Date Range A string used for the range dropdown's default text none
timezoneDescription : string Default value: Select a Timezone A string used for the timezone dropdown's default text none
timezones : Array An array of objects each with a label and value property. Label is displayed in the dropdown and when present value is used for the timezone property exposed on the component. none

endLabel : string Default value: end The label for the lefthand section of the picker when in dual mode. Typically this is the end date. none
